HIGH-LEVEL PROJECT CHOICE
-------------------------
Chosen project title:
How do momentum and mean reversion behaviors vary across sector ETFs, return horizons, and volatility regimes?

WHY THIS IS THE CHOSEN PROJECT
------------------------------
This idea is strong because:
- it uses approved financial time-series data
- it naturally supports multiple well-labeled visualizations
- it allows for statistical analysis beyond surface-level chart interpretation
- it is easier to explain than a more complex ML-only forecasting project
- it creates room for meaningful insights instead of just running one model
- it aligns closely with the rubric emphasis on EDA, interpretation, and clean modeling

The point is not to prove one universal truth like "momentum always wins" or "mean reversion always wins." The point is to investigate whether different return behaviors appear:
- across different sector ETFs
- across different lookback/prediction horizons
- across different volatility regimes

This leads to a more nuanced, research-style conclusion.

PROJECT QUESTION
----------------
Primary question:
How do momentum and mean reversion behaviors differ across sector ETFs, time horizons, and volatility regimes?

Secondary questions:
1. Are short-horizon returns more mean-reverting while medium-horizon returns show more momentum?
2. Do some sectors display stronger trend-following behavior than others?
3. Does high market volatility weaken momentum or strengthen mean reversion?
4. Are the observed patterns stable over time or only present in specific market conditions?

PROJECT SCOPE
-------------
We are NOT building a full trading system.
We are NOT trying to maximize PnL.
We ARE doing a financial time-series research project with:
- cleaning
- visualization
- statistical testing
- basic predictive modeling
- interpretation

-----------
Main assets:

Why these:
- SPY gives a broad market benchmark

If VIX is difficult to obtain cleanly, use rolling realized volatility from SPY returns instead.

Suggested source:

TIME PERIOD
-----------
Use a reasonably long historical window, such as:
- 2015-01-01 to present
or
- 2010-01-01 to present if data quality is good and visuals remain readable

A longer window helps capture multiple market regimes.

CORE RESEARCH DESIGN
--------------------
The project should compare behavior across three dimensions:

1. Asset dimension
   Compare multiple ETFs

2. Horizon dimension
   Compare multiple lookback and forward windows such as:
   - short horizon: 1-day lookback vs next 1-day return
   - medium horizon: 5-day lookback vs next 5-day return
   - longer horizon: 20-day lookback vs next 20-day return

3. Regime dimension
   Split the sample into high-volatility and low-volatility regimes
   Suggested method:
   - compute rolling 20-day volatility for SPY
   - define high-vol regime as above median or above 75th percentile
   - define low-vol regime as below median or below 25th percentile

This gives the final project a strong structure.

PHASE 5 - FEATURE ENGINEERING
-----------------------------
GOAL:
Create the variables needed to study momentum and mean reversion.

WHAT TO DO:
For each ETF:
1. Compute lagged returns for multiple horizons:
   - past 1-day return
   - past 5-day return
   - past 20-day return
2. Compute future returns for matching horizons:
   - next 1-day return
   - next 5-day return
   - next 20-day return
3. Compute rolling volatility:
   - use 20-day rolling standard deviation of daily returns
4. Build a volatility regime indicator using SPY rolling volatility:
   - high-vol regime
   - low-vol regime
5. Optionally compute moving-average gaps or additional context features

WHY WE ARE DOING THIS:
This phase turns raw prices into research variables that let us test whether past moves predict future continuation or reversal.

REQUIRED OUTPUTS:
- engineered feature dataframe for each ETF or one long-form combined dataframe
- clear labeling of lookback variables and forward-return targets
- volatility regime labels

IMPORTANT:
Avoid lookahead bias.
The code must clearly explain:
- what counts as a lagged feature
- what counts as a future target
- why the shifting direction matters

STOP CONDITION:
Do not analyze relationships until all features are correctly aligned.
